# ArchGuard - Командный центр для археологических объектов

**Языки:** [English](README.md) | [Русский](README.ru.md)

Комплексная система мониторинга и безопасности для археологов, работающих в сложных условиях, таких как пещеры. ArchGuard предоставляет мониторинг телеметрии в реальном времени, обнаружение опасностей, трансляцию видео в прямом эфире и обнаружение объектов на основе ИИ с централизованной панели управления командным центром.

## Обзор

ArchGuard — это полнофункциональное приложение, разработанное для мониторинга команд археологов, оснащенных интеллектуальными шлемами с датчиками IMU. Система предоставляет:

- **Мониторинг телеметрии в реальном времени**: отслеживание данных акселерометра и гироскопа от полевых работников
- **Обнаружение опасностей**: автоматические оповещения о падениях, ненормальной вибрации, неправильной осанке и неактивности
- **Трансляция видео в прямом эфире**: получение видеопотоков от камер шлема на базе Raspberry Pi
- **Локальное обнаружение ИИ**: запуск обнаружения объектов YOLO26X на командном центре для анализа сцены в реальном времени
- **Поддержка нескольких языков**: включены интерфейсы на английском и русском языках
- **Управление пользователями**: управление и привязка устройств археологов к системе

## Архитектура системы

### Компоненты

#### Интерфейс (`app.jsx`)
- **Фреймворк**: React с Three.js для 3D визуализации
- **Библиотека пользовательского интерфейса**: Lucide React для иконок
- **Возможности**:
  - Отображение телеметрии в реальном времени (ориентация, ускорение, вращение)
  - Средство просмотра видеопотока в прямом эфире
  - Лента оповещений об опасностях
  - Панель управления пользователями
  - Переключение детектора YOLO и статус
  - Отслеживание времени для работы в пещерах
  - Многоязычный интерфейс (EN, RU)

#### Backend - сервис датчиков и потоковой передачи (`archguard.py`)
- **Фреймворк**: Flask с поддержкой CORS
- **Интеграция I2C**: считывает телеметрию с датчиков IMU MPU6050/6500 на Raspberry Pi
- **Возможности**:
  - Захват данных датчика на частоте 160 Гц
  - Продвинутая фузия датчиков (дополнительная фильтрация)
  - Алгоритмы обнаружения опасностей (удар, неактивность, осанка)
  - Трансляция видео MJPEG/WebRTC с низкой задержкой
  - Зондирование USB-камер и автоматическое обнаружение
  - Регистрация сервиса Zeroconf mDNS

#### Локальный сервис ИИ (`yolo.py`)
- **Фреймворк**: Flask с поддержкой YOLOv8/YOLOv26X
- **Поддержка GPU**: Apple Silicon (MPS), CUDA и режимы CPU
- **Возможности**:
  - Конвейер обнаружения объектов с низкой задержкой
  - Настраиваемые интервал вывода и порог уверенности
  - Вывод потока JPEG из аннотированных кадров
  - Поддержка многих URL-адресов источников (безопасный откат)
  - Валидация кадров прогрева
  - Непрерывный захват кадров из источника камеры

## Установка

### Требования

- **Python 3.8+** (для фоновых сервисов)
- **Node.js/npm** (для разработки интерфейса, если требуется)
- **Оборудование**:
  - Raspberry Pi (для получения данных датчиков и трансляции камер)
  - Датчик IMU (MPU6050 или MPU6500)
  - USB-камера или модуль камеры CSI
  - Компьютер командного центра с GPU (опционально, но рекомендуется)

### Установка Backend

1. **Установите зависимости Python**:
   ```bash
   pip install flask flask-cors smbus2 zeroconf opencv-python ultralytics torch
   ```

2. **Настройка для Mac** (для локального сервиса YOLO):
   ```bash
   # PyTorch с поддержкой MPS для Apple Silicon
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
   ```

3. **Проверьте модель YOLO**: убедитесь, что `yolo26x.pt` или `yolo26x.mlpackage` находится в каталоге проекта

### Установка интерфейса

В настоящее время интерфейс работает через встроенный Babel в `index.html`. Для разработки:

1. Убедитесь в наличии современного веб-браузера с поддержкой модулей ES6
2. Обслуживайте проект на локальном веб-сервере:
   ```bash
   python -m http.server 8000
   ```

## Запуск приложения

### 1. Запустите сервис телеметрии и потоковой передачи
Запустите это на Raspberry Pi или на устройстве с датчиком:
```bash
python archguard.py
# Прослушивает http://localhost:5000
# Предоставляет /video (поток MJPEG), /status (JSON телеметрия) и другие конечные точки
```

### 2. Запустите локальный сервис обнаружения YOLO
Запустите это на компьютере командного центра:
```bash
python yolo.py
# Прослушивает http://localhost:8001
# Конечные точки: /yolo/toggle, /yolo/video, /yolo/status
```

### 3. Откройте панель управления
- Откройте `index.html` в веб-браузере или обслуживайте его через HTTP
- Подключитесь к IP-адресу Raspberry Pi или используйте `arch-helmet.local`, если настроен Zeroconf
- Мониторьте телеметрию в реальном времени, видеопотоки и оповещения об опасностях

## Конфигурация

### Переменные окружения

#### archguard.py
- `PORT`: порт сервера Flask (по умолчанию: 5000)
- `I2C_SENSOR_ADDRESSES`: I2C адреса датчиков IMU (по умолчанию: [0x68, 0x69])
- `USB_CAMERA_INDEXES`: индексы USB-камер для зондирования (по умолчанию: [0, 1, 2, 3])
- `IMPACT_G_THRESHOLD`: порог G-силы для обнаружения падения (по умолчанию: 2,5)
- `STATIONARY_TIME`: время в секундах для обнаружения неактивности (по умолчанию: 5,0)
- `SENSOR_LOOP_HZ`: частота получения датчика (по умолчанию: 160 Гц)

#### yolo.py
- `YOLO_LOCAL_PORT`: порт сервиса (по умолчанию: 8001)
- `YOLO_CONFIDENCE`: порог уверенности обнаружения (по умолчанию: 0,35)
- `YOLO_INFERENCE_INTERVAL_S`: частота вывода в секундах (по умолчанию: 0,0 = непрерывно)
- `YOLO_DEVICE`: устройство для вывода - "mps" (Apple Silicon), "cuda" (NVIDIA) или "" (CPU)
- `YOLO_MODEL_PATH`: пользовательский путь к файлу модели YOLO
- `YOLO_STREAM_JPEG_QUALITY`: качество сжатия JPEG (по умолчанию: 75)
- `ARCHGUARD_STREAM_URL`: пользовательский URL сервиса телеметрии (открывается по умолчанию)

## Конечные точки API

### Сервис телеметрии (archguard.py)

- **GET /status**: Текущее состояние телеметрии (ориентация, ускорение, гироскоп, опасности)
- **GET /video**: Поток видео MJPEG с камеры шлема
- **POST /users**: добавление или управление пользователями
- **GET /health**: проверка работоспособности сервиса

### Сервис обнаружения YOLO (yolo.py)

- **POST /yolo/toggle**: включение/отключение локального обнаружения AI
- **GET /yolo/video**: поток JPEG с аннотированными обнаружениями
- **GET /yolo/status**: статус сервиса и статистика обнаружения
- **GET /yolo/detections**: последние результаты обнаружения (JSON)

## Обнаружение опасностей

Система контролирует следующие опасности:

1. **Обнаружение удара/падения**: срабатывает при ускорении, превышающем `IMPACT_G_THRESHOLD` (2,5G по умолчанию)
2. **Обнаружение неактивности**: срабатывает после `STATIONARY_TIME` секунд минимального движения
3. **Ненормальная вибрация**: обнаруживается через высокочастотную дисперсию гироскопа
4. **Неправильная осанка**: обнаруживается через отклонения тангажа/крена акселерометра
5. **Системные опасности**: пользовательские опасности, обнаруженные ИИ из YOLO

## Особенности архитектуры

### Обработка с низкой задержкой
- **Захват датчика**: многопоточный захват на частоте 160 Гц с дополнительной фильтрацией
- **Вывод YOLO**: непрерывный захват кадров с выводом только на последний кадр
- **Потоковая передача видео**: отдельные потоки для захвата, аннотирования и кодирования MJPEG

### Надежность
- **Фузия датчиков**: дополнительный фильтр объединяет акселерометр и гироскоп для стабильной ориентации
- **Безопасные откаты**: несколько кандидатов устройств камеры и вариантов URL потоковой передачи
- **Дезинфекция**: полезные нагрузки телеметрии проверяются перед обновлениями состояния, чтобы предотвратить сбои

### Масштабируемость
- **Поддержка многоустройства**: система управления пользователями поддерживает привязку нескольких устройств шлема
- **Модульная архитектура**: сервисы взаимодействуют через HTTP, позволяя распределенное развертывание
- **Настраиваемый вывод**: интервал вывода YOLO и уверенность могут быть настроены для производительности

## Устранение неполадок

### Камера не найдена
- Зондирование подключенных USB-камер:
  ```bash
  ls -la /dev/video*
  ```
- Проверьте разрешения на устройство (может потребоваться `sudo` или `usermod`)
- Убедитесь, что камера не используется другим процессом

### Датчик I2C не обнаружен
- Список устройств I2C:
  ```bash
  i2cdetect -y 1  # Используйте 0 для более старых моделей Pi
  ```
- Проверьте подключение датчика и адрес (контакты SDA/SCL)
- Проверьте, включен ли I2C в параметрах Raspberry Pi

### Сервис YOLO не запустится
- Проверьте наличие `yolo26x.pt` в каталоге проекта
- Проверьте установку PyTorch для вашего типа устройства (MPS, CUDA или CPU)
- Просмотрите переопределения переменных окружения

### Интерфейс не подключается
- Убедитесь, что `archguard.py` запущен и доступен по настроенному URL
- Проверьте консоль браузера на предмет ошибок CORS
- Убедитесь, что брандмауэр разрешает подключения на портах 5000 и 8001

## Примечания по разработке

- **Полезные нагрузки телеметрии**: всегда дезинфицируйте поля `orientation`, `accel`, `gyro` и `hazards` для обработки частичных обновлений
- **App.jsx в настоящее время не используется**: источник времени выполнения — это встроенный JSX, скомпилированный Babel в `index.html`
- **Автоматическое восстановление камеры**: инициализация камеры включает валидацию кадров прогрева и автоматическое повторное инициирование при сбое чтения
- **Поддержка формата модели**: YOLO поддерживает как форматы `.pt` (PyTorch), так и `.mlpackage` (CoreML)
